Java模拟高速客运站

发表于 2015-08-10   |   分类于 Java

问题描述

1.1 问题域概述

用面向对象方法和面向对象程序设计语言，实现满足下述要求的一个高速公路客运活动仿真程序。西安市到宝鸡市之间是我省主要的高速公路客运路线之一，经过简化后的客运路线端点、中途停靠点和里程如下图所示（括号里是简称，里程的单位是公里）

1.2 限定条件

（1）从XN 始发至BJ 的客车和从BJ 始发至XN 的客车均有两种车型：沃尔沃（限定乘客人数为48 人）；依维柯（限定乘客人数为21 人）。沃尔沃的速度为1.9 公里/分钟，依维柯的速度为1.4 公里/分钟。
（2）起始状态时，XN 拥有沃尔沃和依维柯客车分别为XNW和XNY 辆，BJ 拥有沃尔沃和依维柯客车分别为BJW和BJY 辆。
（3）从XN 至BJ 和从BJ 至XN 的沃尔沃，均为上午8:30 开始，每小时一班，最后一班为下午5:30；从XN 至BJ 和从BJ 至XN 的依维柯，均为上午8:00 开始，每20 分钟一班，最后一班为下午6:00。

（4）从XN 至BJ 的客车到达BJ 后，即成为从BJ 至XN 的客车，排在当时BJ 同类车型的队尾，再按（3）确定发车时间；从BJ 至XN 的客车到达XN 后的规则相同。
（5）只考虑途中只有乘客下车、没有乘客上车的情况。
（6）有乘客下车时，不论方向与车型，停车时间统一为2 分钟。
（7）乘坐从XN 至BJ 客车的乘客，其下车点为XY、XP、WG、CP、GZ 和BJ 的可能性分别为P_XBXY、P_XBXP、P_XBWG、P_XBCP、P_XBGZ 和P_XBBJ。这些可能性之和为1；乘坐从BJ 至XN 客车的乘客，其下车点为GZ、CP、WG、XP、XY 和XN 的可能性分别为P_BXGZ、P_BXCP、P_BXWG、P_BXXP、P_BXXY 和P_BXXN。这些可能性之和为1。

2.1.3须仿真的客运活动

（1）从上午7:30 开始到下午5:59 为止，每分钟分别在XN 和BJ 随机产生去往BJ 和XN 方向的新到达的乘客。每分钟到达的人数范围为0~PN 人。
（2）按照限定条件，随机产生新到达的乘客的目的地。
（3）乘客按到达的先后顺序上最近一辆该车客满则等候下一辆始发的客车。
（4）若客车到达中途停靠站时有乘客在此下车，按限定条件（5）和（6）处理，否则不停车继续行驶。

使用PowerDesigner16反向工程导出的UML类图

运行情况
开发语言为java，开发环境为Eclipse

界面说明

高速公路中开往宝鸡和西安的两个车队显示在列表中，列表的每个条目含义如下：

客车编号-路程-状态（行驶/停车）-乘客数量-当前所在地点

图中红色框框中客车的状态表示编号是依维柯Y3，已经行驶了138.6km，正在行驶中，车中目前有五名乘客，刚停靠过的地点是CP。

系统运行流畅稳定，没有报错。车站中的乘客生成数量也满足系统容量，不会出现等待乘客不断增加而超过车站服务速率的问题。最终会达到稳态。该程序完整的显示了车站、高速公路、汽车的各项指标，并且将高速公路中的行车状况绘制出来，可以直观的看到在途客车的行驶和停车情况。

系统存在的问题是，现实世界中乘客到达服从泊松分布，而程序中则是均匀到达，有待改进。另外高速公路的绘制不够精确。程序中没有将停车时下去了多少名乘客显示出来，以及没有将汽车距离下一站的距离明确标注出来，只能从图中大概看到汽车是否到达站点。
